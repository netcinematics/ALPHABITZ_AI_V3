## PLAN

Create an implementation plan for the given task and save it to specs/.

### Variables

TASK_DESCRIPTINO: $ARGUMENTSS

### WORKFLOW

1. READ relevant files to understand context
2. EVALUATE task description
3. BREAKDOWN into numbered steps
4. IDENTIFY files to create or modify
5. WRITE plan to specs/{task-name}.md

### REPORT

Confirm spec file created with path and summary.
